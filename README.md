# library
Цель работы: управление доступом к совместно используемым данным и
синхронизация процессов, изучение методов защиты критической секции и
разрешения конфликта совместного доступа к общим ресурсам (данным)
несколькими процессами.
Условие: есть место, где хранятся общие файлы (база данных, журнал записей
и т.д.). К файлам имеют доступ два типа пользователей:
Читатели – пользователи, которые читают содержимое файла, не меняя его.
Несколько читателей могут читать файл одновременно.
Писатели – пользователи, которым разрешено изменять файл. Только один
писатель может писать в файл, остальные ждут, пока он завершит процесс. Во
время записи файла чтение должно быть заблокировано.
Требуется разработать программу, обеспечивающую согласованный доступ к
общему ресурсу для чтения и изменения, избежав конфликтных ситуаций.
Напишите программу, обеспечивающую согласованный доступ к общему
ресурсу для чтения и изменения, избежав конфликтных ситуаций.
Программа должна выводить информацию о том, когда читатель начал и
закончил чтение, когда писатель начал запись и завершил её. Проверьте
работоспособность программы на примерах с разными соотношениями числа
читателей и писателей.
